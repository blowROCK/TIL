stable 소팅과 unstable 소팅에 대해



소팅을 할 때 같은 key값을 가진 node들이 소팅 전과 소팅 후에 순서가 뒤바뀌게 되면 그것을 unstable 소팅이라고 하며, 순서가 뒤바뀌지 않는다면 
stable 소팅이라고 한다.

예를 들어 다음의 case를 생각해보자.

ex) 5, 4, 8, 8, 5, 3, 1, 10, 6

위의 case에서 볼 때 5라는 key값을 가진 노드가 2개 8이라는 key값을 가진 노드가 2개라는 것을 확인 할 수 있다.  
다음과 같은 추가적인 정보를 가진 case에 대해 생각해 보자.

ex) 5(1), 4(2), 8(3), 8(4), 5(5), 3(6), 1(7), 10(8), 6(9)



()안의 숫자는 해당 노드가 들어 온 순서를 뜻한다. 만약 소팅을 할 시에 key 값을 기준으로 소팅을 하되, 같은 key 값을 가진 노드는 들어온 순서에 따라 
다시 소팅이 되어야 한다면 어떻게 할 것인가? 이렇게 최종적으로 소팅 된 정보가 바로 stable 된 정보이다. 즉 소팅 된 결과는 다음과 같아야 한다.

ex) 1(7), 3(6), 4(2), 5(1), 5(5), 6(9), 8(3), 8(4), 10(8)
